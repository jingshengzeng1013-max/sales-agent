# 微信客服接口文档

## 接口一：发送模板消息

### 基本信息

| 项目 | 说明 |
|------|------|
| 请求方式 | `POST` |
| 路径 | `/api/customer-service/template/send` |
| Content-Type | `application/json` |
| 功能描述 | 向已关注公众号的用户发送模板消息。**不受48小时客服消息窗口限制，随时可发送。** 适用于订单通知、服务提醒、活动通知等场景。 |

### 请求参数（Request Body）

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| openid | String | **是** | - | 接收消息的用户openid，用户必须已关注该公众号 |
| templateId | String | 否 | 配置文件中的默认值 | 模板消息ID（在微信公众平台 → 模板消息中申请） |
| url | String | 否 | 无 | 用户点击模板消息后跳转的URL地址 |
| miniprogramAppid | String | 否 | 无 | 点击后跳转的小程序AppID（与url二选一） |
| miniprogramPagepath | String | 否 | 无 | 跳转的小程序页面路径 |
| data | Map\<String, String\> | **是** | - | 模板数据，key对应模板中的 `{{xxx.DATA}}` 参数名，value为要填充的内容 |

### 请求示例

```json
POST /api/customer-service/template/send
Content-Type: application/json

{
  "openid": "obdHZ3ABq00vMKnXEMwQfFK-gOaQ",
  "url": "https://www.baidu.com",
  "data": {
    "first": "资源更新通知",
    "keyword1": "项目教程、编程导航已上线",
    "remark": "点击卡片查看详情"
  }
}

```

### 响应参数（ApiResponse）

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int | 错误码，0表示发送成功 |
| errmsg | String | 错误信息描述 |

### 成功响应

```json
{
    "errcode": 0,
    "errmsg": "ok"
}
```

### 失败响应

```json
// 参数校验失败
{
    "errcode": 400,
    "errmsg": "openid不能为空"
}

// 微信接口错误
{
    "errcode": 40003,
    "errmsg": "invalid openid hint: ..."
}
```

### 错误码说明

| errcode | 说明 |
|---------|------|
| 0 | 发送成功 |
| 400 | 参数校验失败（如openid为空、data为空等） |
| 40003 | 无效的openid（用户不存在或格式错误） |
| 43101 | 用户未关注该公众号，无法发送模板消息 |
| 40014 | 不合法的access_token |
| 41030 | template_id不正确或不存在 |
| -1 | 系统内部异常 |

### 注意事项

- 模板消息**不受48小时客服消息窗口限制**，可随时向已关注用户发送
- `templateId` 不传时使用 `application.yml` 中配置的默认模板ID
- `url` 和 `miniprogramAppid` 为跳转目标，二者选一即可；都不传则点击模板消息无跳转
- `data` 中的 key 必须与模板中定义的参数名严格一致（如 `first`、`keyword1`、`remark` 等）

## 接口二：分页查询用户列表

### 基本信息

| 项目 | 说明 |
|------|------|
| 请求方式 | `GET` |
| 路径 | `/api/customer-service/users` |
| Content-Type | 无需设置 |
| 功能描述 | 分页查询公众号关注用户列表，支持按关注状态筛选，返回用户详细信息 |

### 请求参数（Query Params）

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| page | int | 否 | 1 | 页码，从1开始 |
| size | int | 否 | 10 | 每页条数 |
| subscribe | Integer | 否 | 不传=全部 | 关注状态筛选：1-已关注，0-已取关，不传-全部用户 |

### 请求示例

```
# 查询第1页，每页10条，只看已关注用户
GET /api/customer-service/users?page=1&size=10&subscribe=1

# 查询第2页，每页20条，全部用户（含已取关）
GET /api/customer-service/users?page=2&size=20

# 查询已取关用户
GET /api/customer-service/users?subscribe=0
```

### 响应参数（Page\<WeChatUser\>）

返回 Spring Data 标准分页对象，结构如下：

| 参数名 | 类型 | 说明 |
|--------|------|------|
| content | Array\<WeChatUser\> | 当前页的用户数据列表 |
| totalPages | int | 总页数 |
| totalElements | long | 总记录数 |
| number | int | 当前页码（从0开始） |
| size | int | 每页条数 |
| numberOfElements | int | 当前页实际条数 |
| first | boolean | 是否第一页 |
| last | boolean | 是否最后一页 |

### WeChatUser 对象字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 数据库主键ID |
| openid | String | 用户唯一标识（微信公众号内唯一） |
| nickname | String | 用户昵称 |
| sex | Integer | 用户性别：0-未知，1-男，2-女 |
| city | String | 用户所在城市 |
| province | String | 用户所在省份 |
| country | String | 用户所在国家 |
| headImgUrl | String | 用户头像URL |
| subscribeTime | Long | 用户关注时间（微信返回的unix时间戳） |
| subscribeScene | String | 用户关注的场景值（扫码关注时带场景值） |
| subscribe | Integer | 是否关注：0-已取关，1-已关注 |
| unsubscribeTime | LocalDateTime | 取关时间 |
| groupId | String | 用户所在的分组ID |
| tagIdList | String | 用户标签ID列表，逗号分隔 |
| remark | String | 用户备注 |
| unionid | String | 用户unionid（需绑定开放平台才有） |
| language | String | 语言：zh_CN等 |
| lastInteractTime | LocalDateTime | 最后一次与公众号互动的时间（用于判断48小时客服消息窗口） |
| ragflowSessionId | String | RAGFlow会话ID（用户首次对话时创建，后续复用） |
| createTime | LocalDateTime | 记录创建时间 |
| updateTime | LocalDateTime | 记录更新时间 |

### 注意事项

- `page` 从1开始传入，但响应中 `number` 从0开始（Spring Data 标准行为）
- 不传 `subscribe` 参数时返回全部用户（含已关注和已取关）
- 已取关用户仍保留在数据库中，`subscribe=0`，同时记录了 `unsubscribeTime`
- `lastInteractTime` 用于判断48小时客服消息发送窗口，超过48小时无法主动发送客服消息
