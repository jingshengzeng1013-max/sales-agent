# 修改与优化说明

> 生成日期：2026-04-29  
> 说明范围：本轮围绕 Sales Agent 项目做的代码修复、脚本优化、配置调整、测试补充和文档同步。

---

## 1. 修改目标

本轮优化主要解决上一轮梳理中发现的几个会影响运行和维护的问题：

- Web 检索接口传参和检索器方法签名不一致，可能导致接口运行时报错。
- 结构化抽取脚本默认输入和当前爬虫产物格式不一致。
- 部分脚本入口仍保留 Windows 绝对路径，不利于当前 Linux 部署环境。
- 配置文件存在明显敏感信息硬编码风险。
- 部分历史测试受可选爬虫依赖影响，无法在轻量环境下导入运行。
- README 和部署说明中的命令与当前源码状态不完全一致。

---

## 2. 代码修复

### 2.1 修复混合检索参数不匹配

涉及文件：

- `get_data/src/retrieval/retriever.py`
- `get_data/webapp/server_fastapi.py`
- `get_data/tests/retrieval/test_retriever_options.py`

修改内容：

- `DualRetriever.hybrid_search()` 新增参数：
  - `use_vector`
  - `use_bm25`
  - `city`
- 新增 `_matches_filters()`，统一处理省份、城市、公告类型筛选，避免向量召回和 BM25 召回逻辑不一致。
- 当 `use_vector=False` 时，不再强制调用 Embedding 服务或 FAISS 索引。
- 当 `use_bm25=False` 时，跳过 BM25 召回。
- 当两路召回都关闭时，直接返回空结果。
- `server_fastapi.py` 的 `/api/retrieval/search-tenders` 已把 `use_vector/use_bm25` 传给检索器。
- `/api/retrieval/direct-search` 原本已经传入这些参数，现在检索器可以正确接收。

优化效果：

- 修复 `/api/retrieval/direct-search` 运行时可能出现的 `unexpected keyword argument 'use_vector'` 问题。
- 支持后端对接时显式关闭向量检索或 BM25 检索。
- 城市筛选从 API 请求参数贯通到检索器。
- 在没有 FAISS 索引或不希望调用 Embedding 服务时，可以使用 BM25-only 检索。

### 2.2 兼容结构化抽取 JSONL 输入

涉及文件：

- `get_data/src/etl/core/extract_structured.py`
- `get_data/tests/etl/test_extract_input_formats.py`

修改内容：

- 新增 `_load_detail_records()`，兼容读取：
  - JSON 数组文件
  - JSONL 文件
- 默认输入由 `tenders_detail.json` 改为当前项目实际产物 `data/output/crawler/tenders_detail.jsonl`。
- `--from-json` 帮助文案同步改为“JSON/JSONL”。
- `_canonical_detail_url` 改为可选依赖导入。缺少详情爬虫依赖时，结构化抽取的纯输入格式测试仍可导入。

优化效果：

- 当前爬虫详情产物可以直接进入结构化抽取流程。
- 不需要先手工把 JSONL 转成 JSON 数组。
- 抽取模块对 crawler 依赖的耦合降低。

### 2.3 修复附件 UUID 提取兼容性

涉及文件：

- `get_data/src/crawler/crawl_detail.py`
- `get_data/tests/crawler/test_attachment_extract.py`

修改内容：

- 补回历史测试依赖的 `extract_bizdownload_uuid()`。
- 该函数用于提取中央政府采购网常见的 `class="bizDownload"` 附件链接。
- `extract_attachments_for_detail()` 复用该兼容函数。
- `bs4` 和 `curl_cffi` 改为可选导入：
  - 纯 HTML 附件链接解析不再因为缺少这些依赖而导入失败。
  - 真正发起详情爬取时，如果缺少 `curl_cffi`，会给出明确错误。
  - 详情正文解析缺少 `bs4` 时，会降级为基础文本清洗。

优化效果：

- 历史附件提取测试可在当前环境运行。
- 轻量测试环境不需要安装完整爬虫依赖即可验证纯解析函数。

## 3. 脚本与 CLI 优化

### 3.1 向量化脚本跨平台化

涉及文件：

- `get_data/src/vectorization/vectorize_data.py`
- `get_data/src/vectorization/README.md`

修改内容：

- 移除 `D:/sales_agent/...` 写死路径。
- 新增 argparse CLI：

```bash
python src/vectorization/vectorize_data.py --type all
python src/vectorization/vectorize_data.py --type product
python src/vectorization/vectorize_data.py --type tender
```

- 支持自定义输入输出和模型配置：

```bash
python src/vectorization/vectorize_data.py --type tender \
  --tender-input data/output/etl/tenders_structured.jsonl \
  --tender-output data/embedding/tenders_embedded.jsonl \
  --base-url http://<embedding-host>/v1 \
  --model <embedding-model>
```

优化效果：

- Linux、Windows、容器环境都可通过参数运行。
- README 中的命令和脚本真实参数保持一致。

## 4. 配置安全化

涉及文件：

- `get_data/src/config.py`
- `get_data/webapp/DEPLOYMENT.md`
- `docs/项目说明文档.md`

修改内容：

- `BASE_DIR` 改为默认按 `config.py` 所在路径推导，也支持环境变量覆盖：

```bash
export SALES_AGENT_BASE_DIR=/home/sylincom/sales-agent/get_data
```

- DeepSeek、DashScope、Doubao API Key 改为环境变量读取：

```bash
export DEEPSEEK_API_KEY=<your-key>
export DASHSCOPE_API_KEY=<your-key>
export ARK_API_KEY=<your-key>
```

- 代理配置改为环境变量读取：

```bash
export CRAWLER_USE_PROXY=true
export QG_PROXY_API_URL=<proxy-api-url>
export QG_PROXY_USER=<proxy-user>
export QG_PROXY_PASSWORD=<proxy-password>
```

- Embedding 配置支持环境变量：

```bash
export EMBEDDING_BASE_URL=http://<embedding-host>/v1
export EMBEDDING_MODEL=<embedding-model>
export EMBEDDING_DIMENSION=1024
```

优化效果：

- 明显敏感信息不再写死在源码中。
- 更适合通过 systemd、环境变量或 `.env` 管理不同环境配置。
- 降低密钥误提交风险。

## 8. 当前仍需注意

以下内容不是本轮优化的主体，后续仍建议处理：

- `server_fastapi.py` 和 `webapp/router/` 仍存在两套路由组织方式，建议后续统一。
- `src/api/main.py` 与 `webapp/server_fastapi.py` 的定位仍需明确，生产入口目前以 `webapp.server_fastapi:app` 为准。
- 当前 JSONL、SQLite、Embedding、FAISS 索引的数据规模不完全一致，完整检索前建议重建数据链路。
- 工作区仍存在一些删除文件和未跟踪文件；这些状态在本轮修改前已经存在或来自前序整理，提交前需要统一确认是否纳入版本控制。
