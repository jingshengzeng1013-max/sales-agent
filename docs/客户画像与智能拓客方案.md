# 客户画像构建与智能拓客方案

## 1. 背景与目标
在销售情报系统中，核心挑战在于如何从海量的、零散的招标公告中识别出真正的“高价值客户”。通过构建**360°客户画像**，我们可以实现从“被动搜索”到“主动挖掘”的转变，解决“客户是谁、需求是什么、该怎么跟进”的核心问题。

## 2. 数据来源
画像的构建依托于多源数据的融合：
*   **外部数据**：从招标网（CCGP）爬取的原始标讯、详情及附件。
*   **结构化数据**：利用 LLM 提取的 `tender_structured` 数据（包含采购单位、技术需求、预算、中标信息等）。
*   **内部产品数据**：公司自主研发的产品库（`get_data/data/product.jsonl`），包含芯片型号、技术指标、应用场景等。

## 3. 画像维度与标签体系

### 3.1 基础属性 (Who)
*   **地域标签**：省份、城市、行政级别。
*   **行业赛道**：通过单位名称关键词（如“气象局”、“公安局”、“大学”）自动划分所属行业。
*   **活跃度**：历史招标频次、最新招标日期（识别活跃客户 vs 沉睡客户）。

### 3.2 需求属性 (What)
*   **技术偏好**：聚合标讯中的 `product_keywords`，识别客户对特定技术（如：卫星通信、FPGA、高通量）的依赖度。
*   **应用场景**：从 `application_scenario` 识别客户的具体业务领域（如：应急指挥、科研试验）。
*   **采购阶段**：区分意向期（招标公告）与成交期（中标公告）。

### 3.3 价值属性 (Value)
*   **预算实力**：历史累计预算金额、单笔平均金额。
*   **商机评分**：基于标讯内容自动生成的 `opportunity_score`。

### 3.4 竞争属性 (How)
*   **存量供应商**：识别 `winning_bidder`，分析客户的现有合作伙伴。
*   **产品替代倾向**：分析 `winning_product`，评估我司产品（如 DX-S702 芯片）的切入机会。

## 4. 自动化挖掘客户路径

### 4.1 智能商机主动推送 (Push Mode)
*   **逻辑**：每日自动运行匹配脚本。
*   **动作**：将新标讯与产品库进行向量匹配（Faiss + BM25），相似度 > 0.8 时自动通过企微/邮件推送给对应销售。
*   **辅助**：LLM 自动生成推荐理由（如：“该项目对基带处理的高带宽需求与 DX-M DSP 核高度匹配”）。

### 4.2 相似客户挖掘 (Look-alike)
*   **逻辑**：以现有高价值客户为“种子”。
*   **动作**：在全量数据库中搜索行业相同、技术偏好相似、且处于活跃期的潜在单位。

### 4.3 竞争对手存量替换 (Replacement Mining)
*   **逻辑**：监控竞争对手中标记录。
*   **动作**：针对 2-3 年前采购了竞品、且近期有新动态的单位，提醒销售进行“升级换代”式拜访。

### 4.4 自动化销售话术辅助
*   **逻辑**：画像数据 + LLM。
*   **动作**：针对特定标讯，自动生成个性化开发信。
    *   *示例*：“关注到贵司在[应急场景]中的需求，我们星通科技的[产品]曾服务于相似的[标杆客户]...”

## 5. 技术实现架构
1.  **数据层**：SQLite (`ccgp_data.db`) 存储结构化标讯。
2.  **索引层**：`src/vectorization/` 构建 FAISS (向量索引) + BM25 (关键词索引)。
3.  **检索层**：`src/retrieval/retriever.py` 的 `DualRetriever` 类实现 RRF 融合检索。
4.  **应用层**：`src/analysis/` Python 自动化脚本 + Web 管理看板。
