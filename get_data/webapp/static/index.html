招投标数据采集与销售情报匹配系统

📊 销售情报 · 全流程工作台

将业务链路拆成三步，便于对照执行。第 1、2 步在对应标签中操作；第 3 步可点卡片内③打开混合检索标签（产品库 FAISS+BM25；可从库内选招标作查询）。 匹配输入来自 tender_structured（第 2 步产物）。完整销售情报流水线请用脚本或命令行调用 run_sales_intel_flow。

1

数据爬取与入库

政府采购网 列表 → 详情 → output/data/tenders_raw.json → 写入 SQLite（tenders 等）。没有落库的原文，后续结构化与销售情报无从谈起。

  • 打开「数据爬取」，按需勾选列表 / 入库 / 详情；流式日志在页面底部「响应日志」
  • 可选打开「数据库」确认 tenders 是否有新行、content 是否已抓正文
2

结构化处理

从库中读正文，经 LLM 抽取字段 → output/etl/tenders_structured*.json，并可导入 tender_structured。销售情报会优先使用结构化里的 product_keywords、标准买方名等；建议在本步完成抽取再跑第 3 步。

  • 「结构化抽取」中设置条数、模型；可勾选「抽取结束后自动导入数据库」
  • 需要对比不同模型时，用「结果对比」或修改输出后缀生成多份 JSON
3

客户识别 · 产品混合检索 · 排序列表

从库中读取第 2 步已写入的 tender_structured 后，完整流水线（客户识别、产品匹配、排序）请在项目根目录运行 python src/sales_intel_flow.py 等脚本；本页「混合检索」标签仅做产品库检索试跑。

  • ③ 去混合检索：与销售情报共用产品索引，适合快速试检索 ranked 列表

历史快照对比（可选）

默认不再写入快照；若曾保存过 sales_intel_leads*.json，可在此按「采购单位 + 项目名称」对齐对比
