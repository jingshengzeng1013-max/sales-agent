# 附件链接修复说明

## 问题描述

中国政府采购网的招标公告详情页中，附件下载链接的 `<a>` 标签格式如下：

```html
<a class='bizDownload' href='' id='3048A92F3366BCAB13ACDB3164FD57' title='点击下载'>
    文件名.pdf
</a>
```

**问题**：`href=""` 是空的，实际下载 URL 需要通过 `id`（UUID）拼接生成。

之前的代码只提取了 `href` 属性，导致这些附件链接丢失或为空。

## 解决方案

### 1. 新增 `extract_bizdownload_uuid` 函数

从 HTML 中提取所有 `class='bizDownload'` 的 `<a>` 标签，并生成实际下载 URL：

```python
def extract_bizdownload_uuid(page_html):
    """
    中国政府采购网的附件链接格式：
    <a class='bizDownload' href='' id='UUID' title='点击下载'>文件名.pdf</a>

    实际下载 URL 为：https://download.ccgp.gov.cn/oss/download?uuid=UUID
    """
```

**逻辑**：
1. 用正则匹配所有 `<a class='bizDownload'>...</a>` 标签
2. 提取 `id` 属性作为 UUID
3. 提取标签内文本作为文件名
4. 生成实际下载 URL：`https://download.ccgp.gov.cn/oss/download?uuid={UUID}`

### 2. 修改 `extract_attachment_urls` 函数

在提取附件 URL 时，优先调用 `extract_bizdownload_uuid` 生成可访问的下载链接。

### 3. 修改 `parse_detail` 函数

新增 `attachment_details` 字段，保存附件的详细信息（文件名、UUID、下载 URL）。

## 注意事项

1. 生成的下载 URL 域名为 `download.ccgp.gov.cn`，需要确保网络可访问
2. 部分旧公告可能没有 `bizDownload` 格式的附件，仍然依赖普通 `<a href>` 链接
3. 附件下载可能需要 Cookie 或 Referer 验证，直接访问可能返回 403
