# 常见问题排查

本文档汇总系统使用过程中可能遇到的常见问题及其解决方案。

---

## 一、启动问题

### 1.1 端口被占用

**错误信息**：
```
ERROR: Could not bind to port 8103
OSError: [WinError 10048] Only one usage of each socket address is normally permitted
```

**解决方案**：

```powershell
# 1. 查看端口占用
netstat -ano | findstr 8103

# 2. 如果是 Python 进程占用，结束它
taskkill /PID <PID> /F

# 3. 或修改端口（临时）
# 编辑 webapp/server_fastapi.py，修改 PORT = 8104
```

### 1.2 数据库文件不存在

**错误信息**：
```
FileNotFoundError: [Errno 2] No such file or directory: 'data/ccgp_data.db'
```

**解决方案**：

```bash
# 初始化数据库
python src/utils/reset_db.py
```

### 1.3 依赖模块导入失败

**错误信息**：
```
ModuleNotFoundError: No module named 'xxx'
```

**解决方案**：

```bash
# 安装缺失的依赖
pip install -r requirements.txt

# 或单独安装
pip install <module_name>
```

## 二、爬虫问题

### 2.1 爬取被拦截

**错误信息**：
- 页面返回 "频繁访问"
- HTTP 403 Forbidden
- 无数据返回

**解决方案**：

```python
# 1. 增加请求间隔（编辑 src/config.py）
CRAWLER_CONFIG = {
    "delay_min": 2.0,  # 原来 0.5
    "delay_max": 5.0,  # 原来 1.5
}

# 2. 启用代理
CRAWLER_CONFIG = {
    "use_proxy": True,
    "proxy_api_url": "https://share.proxy.qg.net/get?key=YOUR_KEY",
}

# 3. 更换网络环境
```

### 2.2 代理不可用

**错误信息**：
```
ProxyError: 代理连接失败
```

**解决方案**：

```bash
# 测试代理 API
python src/crawler/test_proxy_api.py

# 检查代理余额
# 访问代理服务后台

# 更换代理服务提供商
```

### 2.3 数据为空

**错误信息**：
```
[WARN] 未找到任何招标公告
```

**解决方案**：

```bash
# 1. 检查关键词配置
grep "keyword" src/config.py

# 2. 手动访问网站验证
# https://search.ccgp.gov.cn/bxsearch

# 3. 检查时间范围配置
# timeType: "4" 表示近三月，可能需要扩大范围
```

## 三、数据处理问题

### 3.1 LLM 调用失败

**错误信息**：
```
RateLimitError: Exceeded quota
AuthenticationError: Invalid API Key
```

**解决方案**：

```bash
# 1. 检查 API Key 配置
# 编辑 src/config.py
DEEPSEEK_CONFIG = {
    "api_key": "sk-xxx",  # 确认正确
}

# 2. 检查账户余额
# 访问 DeepSeek/阿里云控制台

# 3. 切换到本地 LLM
LOCAL_LLM_CONFIG = {
    "base_url": "http://10.210.10.51:8001/v1",
}
```

### 3.2 抽取结果为空

**错误信息**：
```
[WARN] 抽取失败: tender_id=123
```

**解决方案**：

```bash
# 1. 检查原始数据
# 查看 logs/extract_*.log 获取详情

# 2. 检查 content 字段是否有内容
sqlite3 data\ccgp_data.db "SELECT id, LENGTH(content) FROM tenders LIMIT 5;"

# 3. 修复内容为空的数据
python src/crawler/crawl_detail.py repair-content
```

### 3.3 Token 消耗过高

**问题**：LLM 调用费用超出预期

**解决方案**：

1. 使用缓存（同一条数据不重复抽取）
2. 使用更便宜的模型（如 qwen-plus 替代 qwen-max）
3. 减少抽取条数
4. 开启请求缓存

```python
# 编辑 src/config.py
DEEPSEEK_CONFIG = {
    "model": "deepseek-chat",  # 切换到更便宜的模型
}
```

## 四、检索问题

### 4.1 检索结果为空

**错误信息**：
无结果返回

**解决方案**：

```bash
# 1. 确认数据库有数据
sqlite3 data\ccgp_data.db "SELECT COUNT(*) FROM tender_structured;"

# 2. 确认索引存在
dir data\index_tenders\

# 3. 检查索引文件大小
dir data\index_tenders\*.index

# 4. 重建索引
python src/vectorization/build_index.py --type tender
```

### 4.2 检索速度慢

**问题**：查询响应时间过长

**解决方案**：

```bash
# 1. 减少返回条数
# top_k 参数从 100 减少到 20

# 2. 清理索引缓存
# 重启服务

# 3. 优化数据库
sqlite3 data\ccgp_data.db "PRAGMA optimize;"
```

### 4.3 索引加载失败

**错误信息**：
```
IndexNotFoundError: Faiss index not found
```

**解决方案**：

```bash
# 重新构建索引
python src/vectorization/vectorize_data.py --type tender
python src/vectorization/build_index.py --type tender
```

## 五、服务问题

### 5.1 服务无响应

**排查步骤**：

```powershell
# 1. 检查进程是否运行
tasklist | findstr python

# 2. 检查端口监听
netstat -ano | findstr 8103

# 3. 查看错误日志
type logs\api_server\*.log | findstr /i error
```

### 5.2 前端无法访问

**排查步骤**：

1. 确认服务已启动
2. 确认端口未被防火墙拦截
3. 尝试访问 API 端点

```powershell
# 测试 API
curl http://127.0.0.1:8103/api/db/tables

# 检查防火墙
netsh advfirewall firewall show rule name="SalesAgent"
```

### 5.3 跨域问题

**错误信息**：
```
Access to fetch at 'http://xxx' from origin 'http://yyy' has been blocked by CORS policy
```

**解决方案**：

```python
# 编辑 webapp/server_fastapi.py
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 生产环境应限制具体域名
)
```

## 六、数据问题

### 6.1 数据库损坏

**错误信息**：
```
sqlite3.DatabaseError: database disk image is malformed
```

**解决方案**：

```bash
# 1. 备份损坏的数据库
copy data\ccgp_data.db data\ccgp_data_corrupted.db

# 2. 导出数据（如果可能）
sqlite3 data\ccgp_data_corrupted.db ".backup data\ccgp_data_new.db"

# 3. 使用备份恢复
copy data\backup\ccgp_data.db data\ccgp_data.db
```

### 6.2 数据不一致

**问题**：索引与数据库数据不匹配

**解决方案**：

```bash
# 重新构建索引
python src/vectorization/build_index.py --type tender --rebuild
```

## 七、日志分析

### 7.1 日志位置

```
logs/
├── api_server/*.log          # API 服务日志
├── extract_*.log             # LLM 抽取日志
├── crawl_*.log               # 爬虫日志
└── stream/*.log              # 流式日志
```

### 7.2 常用日志命令

```powershell
# 搜索错误
findstr /i /c:"error" logs\*.log

# 搜索异常
findstr /i /c:"exception" logs\*.log

# 查看最近 100 行
Get-Content logs\api_server\api_server_latest.log -Tail 100

# 实时监控
Get-Content logs\api_server\api_server_latest.log -Wait
```

## 八、快速诊断清单

当系统出现问题时，按以下顺序检查：

```
□ 1. 服务是否运行？
   - netstat -ano | findstr 8103

□ 2. 依赖服务是否正常？
   - curl http://10.210.10.51:8001/v1/models
   - curl http://10.210.10.51:8022/v1/models

□ 3. 数据库是否有数据？
   - sqlite3 data\ccgp_data.db "SELECT COUNT(*) FROM tender_structured;"

□ 4. 索引是否存在？
   - dir data\index_tenders\

□ 5. 日志有什么错误？
   - findstr /i /c:"error" logs\api_server\*.log
```
