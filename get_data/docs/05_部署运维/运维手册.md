# 运维手册

本文档提供系统日常运维操作的详细指南，包括数据管理、日志查看、服务监控等。

---

## 一、日常运维

### 1.1 启动服务

```bash
# 进入项目目录
cd D:\sales_agent\get_data

# 启动 FastAPI 服务
python webapp/server_fastapi.py

# 服务启动后会显示：
# Uvicorn running on http://0.0.0.0:8103
# 访问地址：http://127.0.0.1:8103/static/demo.html
```

### 1.2 停止服务

```powershell
# Windows 查看进程
netstat -ano | findstr 8103

# 结束进程
taskkill /PID <PID> /F
```

### 1.3 服务自启动（Windows）

创建批处理文件 `start_server.bat`：

```batch
@echo off
cd /d D:\sales_agent\get_data
start "SalesAgent" python webapp/server_fastapi.py
```

将快捷方式放入启动目录：
```powershell
shell:startup
```

## 二、数据管理

### 2.1 数据备份

```bash
# 备份数据库
copy data\ccgp_data.db data\backup\ccgp_data_%date:~0,4%%date:~5,2%%date:~8,2%.db

# 备份所有数据
xcopy /E /I /Y data\output data\backup\output_%date:~0,4%%date:~5,2%%date:~8,2%

# 备份索引（可选，索引较大）
xcopy /E /I /Y data\index_tenders data\backup\index_tenders_%date:~0,4%%date:~5,2%%date:~8,2%
```

### 2.2 数据恢复

```bash
# 恢复数据库
copy data\backup\ccgp_data_20260420.db data\ccgp_data.db

# 恢复输出文件
xcopy /E /I /Y data\backup\output_20260420 data\output\
```

### 2.3 数据清理

```bash
# 清理旧的 HTML 文件（可释放大量空间）
del /Q data\html\*.html

# 清理过期日志（保留30天）
forfiles /p "logs" /s /d -30 /c "cmd /c del /q @path"

# 清理空目录
for /f "delims=" %%d in ('dir /ad /b /s data\html 2^>nul') do rmdir "%%d" 2>nul
```

## 三、日志管理

### 3.1 日志目录结构

```
logs/
├── api_server/           # API 服务日志
│   └── api_server_20260420_143025.log
├── extract_*.log         # LLM 抽取日志
├── crawl_*.log           # 爬虫日志
└── stream/               # 流式日志
```

### 3.2 查看日志

```bash
# 查看最新 API 日志
dir /o:-d /b logs\api_server\*.log | select -first 1
type logs\api_server\api_server_20260420_143025.log

# 实时查看日志（Windows PowerShell）
Get-Content -Path logs\api_server\api_server_20260420_143025.log -Wait

# 搜索错误日志
findstr /i /c:"error" logs\api_server\*.log
```

### 3.3 日志轮转

系统日志达到 100MB 时会自动轮转，历史日志保留 7 天。

## 七、故障恢复

### 7.1 服务无法启动

1. 检查端口是否被占用：`netstat -ano | findstr 8103`
2. 检查日志：`type logs\api_server\*.log`
3. 常见问题：
   - 索引文件损坏 → 重建索引
   - 数据库锁定 → 删除 .wal 文件
   - 依赖服务离线 → 检查 Embedding/LLM 服务

### 7.2 检索结果为空

```bash
# 1. 确认数据库有数据
sqlite3 data\ccgp_data.db "SELECT COUNT(*) FROM tender_structured;"

# 2. 确认索引存在
dir data\index_tenders\

# 3. 如索引损坏，重建
python src/vectorization/build_index.py --type tender
```

### 7.3 LLM 调用失败

```bash
# 1. 检查 LLM 服务
curl http://10.210.10.51:8001/v1/models

# 2. 检查 API Key 配置
# 编辑 src/config.py

# 3. 查看详细错误日志
findstr /i /c:"error" /c:"exception" logs\*.log
```

## 八、安全建议

### 8.1 API Key 保护

- 不要将 API Key 提交到代码仓库
- 使用环境变量存储敏感信息
- 定期更换 API Key

### 8.2 网络访问控制

- 生产环境限制 `allow_origins`
- 使用防火墙限制外部访问
- 定期检查访问日志

### 8.3 数据备份

- 建议每日自动备份
- 备份文件异地存储
- 定期测试恢复流程
