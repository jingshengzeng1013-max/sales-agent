# 部署指南

本文档详细介绍系统的部署流程，包括开发环境、生产环境部署以及相关配置说明。

---

## 一、环境要求

### 1.1 硬件要求

| 资源 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 2 核 | 4 核及以上 |
| 内存 | 4 GB | 8 GB 及以上 |
| 磁盘 | 10 GB 可用空间 | 20 GB 及以上 |
| 网络 | 可访问政府采购网 | 专线或企业宽带 |

### 1.2 软件要求

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| Python | 3.10+ | 建议使用 3.10 或 3.11 |
| pip | 最新版 | Python 包管理器 |
| SQLite | 3.x | 已随 Python 内置 |

### 1.3 外部依赖

系统依赖以下外部服务：

| 服务 | 必需 | 说明 |
|------|------|------|
| 中国政府采购网 | 是 | 数据源 |
| Embedding 服务 | 是 | 文本向量化（`http://10.210.10.51:8022/v1`） |
| 本地 LLM 服务 | 是 | AI 销售建议生成（`http://10.210.10.51:8001/v1`） |
| DeepSeek API | 否 | 云端 LLM 备选 |
| 阿里云 DashScope | 否 | 通义千问 API |

## 三、部署步骤

### 3.1 环境准备

```bash
# 1. 创建并进入项目目录
cd D:\sales_agent\get_data

# 2. 创建 Python 虚拟环境（推荐）
python -m venv venv

# 3. 激活虚拟环境
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# 4. 安装依赖
pip install -r requirements.txt
```

### 3.2 配置文件

编辑 `src/config.py`，根据实际环境配置：

```python
# 数据库路径
DB_PATH = DATA_DIR / "ccgp_data.db"

# 爬虫关键词
CRAWLER_CONFIG = {
    "keyword": "通信",  # 根据业务需求修改
    "max_pages": 100,
    ...
}

# 本地 LLM（根据实际部署地址修改）
LOCAL_LLM_CONFIG = {
    "base_url": "http://10.210.10.51:8001/v1",
    "model": "/models/Qwen3.5-27B",
}

# Embedding 服务（根据实际部署地址修改）
EMBEDDING_CONFIG = {
    "base_url": "http://10.210.10.51:8022/v1",
    "dimension": 1024,
}
```

### 3.3 初始化数据库

```bash
# 重置数据库（如已有数据可跳过）
python src/utils/reset_db.py
```

### 3.4 构建索引

首次部署或数据更新后需要构建索引：

```bash
# 1. 向量化数据
python src/vectorization/vectorize_data.py --type tender

# 2. 构建索引
python src/vectorization/build_index.py --type tender
```

## 四、服务部署

### 4.1 开发环境

```bash
# 启动 FastAPI 服务
python webapp/server_fastapi.py

# 访问地址
# http://127.0.0.1:8103/static/demo.html
# http://127.0.0.1:8103/static/index.html
```

### 4.2 生产环境

#### 使用 uvicorn 直接运行

```bash
# 后台运行
nohup uvicorn webapp.server_fastapi:app --host 0.0.0.0 --port 8103 --log-file logs/uvicorn.log > /dev/null 2>&1 &

# 或使用 systemd 服务管理（Linux）
```

#### 使用 gunicorn（Linux）

```bash
# 安装 gunicorn
pip install gunicorn

# 启动服务
gunicorn -w 4 -b 0.0.0.0:8103 webapp.server_fastapi:app --log-file logs/gunicorn.log
```

#### 使用 Windows 服务

可以使用 NSSM 将 FastAPI 注册为 Windows 服务：

```powershell
# 下载 nssm
# 创建服务
nssm install SalesAgent "python" "webapp\server_fastapi.py"
nssm set SalesAgent AppDirectory "D:\sales_agent\get_data"
nssm set SalesAgent DisplayName "Sales Agent Service"
nssm set SalesAgent Description "招投标数据系统服务"
nssm start SalesAgent
```

## 六、数据迁移

### 6.1 迁移数据库

```bash
# 备份
copy data\ccgp_data.db data\backup\ccgp_data_%date:~0,4%%date:~5,2%%date:~8,2%.db

# 迁移
# 直接复制 db 文件到新服务器对应目录
```

### 6.2 迁移索引

```bash
# 备份索引
xcopy /E /I data\index data\backup\index
xcopy /E /I data\index_tenders data\backup\index_tenders
```

### 6.3 完整迁移清单

| 目录/文件 | 说明 | 必需 |
|-----------|------|------|
| `data/ccgp_data.db` | SQLite 数据库 | 是 |
| `data/index/` | 产品向量索引 | 是 |
| `data/index_tenders/` | 招标向量索引 | 是 |
| `data/output/` | 输出文件 | 否 |
| `src/config.py` | 配置文件 | 是 |
| `logs/` | 日志文件 | 否 |

## 八、常见部署问题

### 8.1 端口被占用

```powershell
# 查看端口占用
netstat -ano | findstr 8103

# 结束进程或修改端口
```

### 8.2 数据库锁定

```bash
# 关闭所有连接后重试
# 或删除 wal 文件
del data\ccgp_data.db-wal
del data\ccgp_data.db-shm
```

### 8.3 索引加载失败

```bash
# 重新构建索引
python src/vectorization/build_index.py --type all
```
