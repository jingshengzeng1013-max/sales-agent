# 日志系统使用说明

## 目录结构

```
D:\get_data\
├── logs/                      # 统一日志目录
│   ├── api_server_*.log       # API 服务器日志
│   ├── web_app_*.log          # Web 前端日志
│   ├── extract_structured_*.log  # 数据抽取日志
│   ├── crawler_*.log          # 爬虫日志
│   └── ...                    # 其他模块日志
│
└── src/
    └── utils/
        ├── logger.py          # 日志配置模块
        └── manage_logs.py     # 日志管理工具
```

## 快速使用

### 查看日志

```bash
# 查看最近 7 天的日志
python src/utils/manage_logs.py list --days 7

# 查看最近 30 天的日志
python src/utils/manage_logs.py list --days 30
```

### 清理日志

```bash
# 预览将删除哪些日志（默认保留 30 天）
python src/utils/manage_logs.py clean --days 30

# 执行删除
python src/utils/manage_logs.py clean --days 30 --execute
```

### 查看日志内容

```bash
# 查看日志文件末尾 50 行
python src/utils/manage_logs.py tail extract_structured_20260401_171911.log

# 查看日志文件末尾 100 行
python src/utils/manage_logs.py tail extract_structured_20260401_171911.log --lines 100
```

### Windows 用户快捷方式

```bash
# 日志管理菜单
logs_manager.bat
```

## 在代码中使用日志

```python
from utils.logger import setup_logger

# 初始化日志
logger = setup_logger('my_module', log_to_file=True)

# 使用日志
logger.debug("调试信息")
logger.info("一般信息")
logger.warning("警告信息")
logger.error("错误信息")
logger.critical("严重错误")
```

## 日志格式

```
2026-04-01 17:53:56 | INFO     | module_name | 日志内容
```

| 字段 | 说明 |
|------|------|
| `2026-04-01 17:53:56` | 时间戳 |
| `INFO` | 日志级别 |
| `module_name` | 模块名称 |
| `日志内容` | 日志消息 |

## 日志级别

| 级别 | 数值 | 说明 | 输出位置 |
|------|------|------|----------|
| DEBUG | 10 | 调试信息 | 文件 + 控制台 |
| INFO | 20 | 一般信息 | 文件 + 控制台 |
| WARNING | 30 | 警告信息 | 文件 + 控制台 |
| ERROR | 40 | 错误信息 | 文件 + 控制台 |
| CRITICAL | 50 | 严重错误 | 文件 + 控制台 |

## 日志文件命名规则

```
{模块名}_{YYYYMMDD}_{HHMMSS}.log
```

示例：
- `api_server_20260401_175356.log` - API 服务器日志
- `extract_structured_20260401_161732.log` - 数据抽取日志

## API 请求日志

API 服务器会自动记录每个请求：

```
2026-04-01 17:55:17 | INFO     | api_server | GET /api/stats/overview - 开始处理
2026-04-01 17:55:17 | INFO     | api_server | GET /api/stats/overview - 状态：200 - 耗时：0.002s
```

包含信息：
- 请求方法和路径
- 处理状态
- 响应状态码
- 执行耗时

## 自动清理建议

建议定期清理旧日志，避免占用过多磁盘空间：

```bash
# 每周清理一次，保留最近 30 天的日志
0 2 * * 0 python src/utils/manage_logs.py clean --days 30 --execute
```

## 日志目录配置

修改 `src/utils/logger.py` 中的 `LOGS_DIR` 变量可以改变日志存储位置：

```python
# 默认日志目录
LOGS_DIR = os.path.join(BASE_DIR, "logs")
```
