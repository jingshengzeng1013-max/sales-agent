# 代码规范

本文档规定项目的代码编写规范，确保代码质量和一致性。

---

## 一、Python 代码规范

### 1.1 编码声明

所有 Python 文件头部必须包含：

```python
# -*- coding: utf-8 -*-
```

### 1.2 导入规范

```python
# 标准库
import os
import sys
import json
from typing import Dict, Any, Optional

# 第三方库
import requests
from openai import OpenAI

# 本地模块（使用绝对导入）
from src.config import DB_PATH
from src.utils.jsonl_helper import load_jsonl
```

### 1.3 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 模块名 | 小写下划线 | `sales_advisor.py` |
| 类名 | 大驼峰 | `SalesAdvisor`, `DualRetriever` |
| 函数名 | 小写下划线 | `generate_suggestions()` |
| 常量 | 全大写下划线 | `MAX_PAGES`, `DEFAULT_TIMEOUT` |
| 变量名 | 小写下划线 | `project_data`, `customer_name` |

### 1.4 函数文档

```python
def generate_suggestions(project_data: Dict[str, Any], customer_profile: Optional[Dict] = None) -> str:
    """
    生成销售切入点建议

    Args:
        project_data: 项目数据字典，包含项目名称、采购单位、预算等信息
        customer_profile: 可选的客户画像数据

    Returns:
        返回 Markdown 格式的销售建议字符串

    Raises:
        ValueError: 当项目数据格式不正确时抛出
    """
```

## 二、API 设计规范

### 2.1 RESTful 风格

| 操作 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 列表 | GET | `/api/retrieval/search-tenders` | 搜索招标 |
| 详情 | GET | `/api/customer/profile/{name}` | 获取画像 |
| 创建 | POST | `/api/analysis/sales-suggestions` | 生成建议 |

### 2.2 请求/响应格式

**请求体**：
```json
{
    "product_id": "string",
    "top_k": 20,
    "options": {}
}
```

**成功响应**：
```json
{
    "code": 200,
    "data": {},
    "message": "success"
}
```

**错误响应**：
```json
{
    "code": 400,
    "data": null,
    "message": "Invalid parameter: product_id is required"
}
```

## 三、数据库规范

### 3.1 表命名

- 使用小写字母
- 使用下划线分隔
- 名称要体现业务含义

### 3.2 字段命名

```sql
-- 推荐
project_name, buyer_name, created_at

-- 避免
projectName, buyerName, createdAt (驼峰命名)
```

### 3.3 SQL 编写

```python
# 使用参数化查询，避免 SQL 注入
cursor.execute("SELECT * FROM tenders WHERE id = ?", (tender_id,))

# 避免 SELECT *
cursor.execute("SELECT id, project_name, publish_date FROM tenders")
```

## 四、日志规范

### 4.1 日志级别

| 级别 | 使用场景 |
|------|----------|
| DEBUG | 调试信息，详细流程 |
| INFO | 正常业务流程 |
| WARNING | 警告，但不影响功能 |
| ERROR | 错误，需要调查 |
| CRITICAL | 严重错误，系统不可用 |

### 4.2 日志格式

```python
import logging

logger = logging.getLogger(__name__)

# 格式：[时间] [级别] [模块] 消息
# 2026-04-20 10:30:00 INFO sales_advisor 正在生成建议...
```

### 4.3 日志记录

```python
# 好的实践
logger.info(f"正在为项目 [{project_name}] 生成 AI 销售建议...")
logger.warning(f"缓存未命中，将调用 LLM")
logger.error(f"生成失败: {e}", exc_info=True)

# 避免
print("info message")  # 使用 print 调试，生产用 logger
logger.debug(f"用户输入: {user_input}")  # 避免记录敏感信息
```

## 五、错误处理规范

### 5.1 异常捕获

```python
try:
    result = some_function()
except ValueError as e:
    logger.error(f"参数错误: {e}")
    raise
except Exception as e:
    logger.exception("未知错误")
    return None
```

### 5.2 自定义异常

```python
class DataNotFoundError(Exception):
    """数据未找到异常"""
    pass

class ValidationError(Exception):
    """数据验证异常"""
    pass
```

## 九、安全规范

### 9.1 敏感信息

- 禁止硬编码 API Key、密码
- 使用环境变量或配置文件
- 日志中不要记录敏感信息

```python
# 错误
api_key = "sk-1234567890abcdef"

# 正确
api_key = os.environ.get("DEEPSEEK_API_KEY")
```

### 9.2 用户输入

- 验证所有用户输入
- 使用参数化查询
- 转义 HTML 防止 XSS

## 十、注释规范

### 10.1 必要注释

- 复杂业务逻辑
- 非显而易见的代码
- 重要的决策说明

### 10.2 注释格式

```python
# 单行注释使用 # 号
# 这是单行注释

def complex_function():
    """
    多行文档字符串使用三引号

    用于函数、类的详细说明
    """
```

### 10.3 TODO 标记

```python
# TODO(username): 待完成功能说明
# FIXME(username): 需要修复的问题
```
