### 2.1 推荐方案：分层向量化

```
┌─────────────────────────────────────────────────────────────┐
│                    向量化策略                                │
├─────────────────────────────────────────────────────────────┤
│  【核心向量】(必需)                                          │
│  product_keywords + application_scenario                    │
│  + technical_requirements_summary                           │
│  → 用于主检索                                                │
├─────────────────────────────────────────────────────────────┤
│  【采购人向量】(可选)                                        │

│  → 用于找相似客户/历史中标方                                 │
├─────────────────────────────────────────────────────────────┤
│  【不过滤字段】                                              │
│  province, city, budget_amount, announce_type               │

└─────────────────────────────────────────────────────────────┘
```

### 3.2 检索流程

```
                    用户查询
                       │
                       ▼
        ┌──────────────────────────┐
```

## 四、技术选型

### 4.1 嵌入模型（Embedding Model）

| 模型 | 维度 | 中文效果 | 推荐度 |
|------|------|---------|--------|
| `text-embedding-3-small` (OpenAI) | 1536 | ★★★★☆ | 推荐 |
| `bge-m3` (智源) | 1024 | ★★★★★ | 强烈推荐 |
| `m3e-base` | 768 | ★★★★☆ | 推荐 |
| `text2vec-large-chinese` | 768 | ★★★☆☆ | 可用 |

**推荐**: `bge-m3` 或 `m3e-base`（开源、中文效果好、可本地部署）

### 4.2 向量存储方案

| 方案 | 适用规模 | 优点 | 缺点 |
|------|---------|------|------|
| **SQLite + NumPy** | <10 万条 | 简单、零依赖 | 大规模性能差 |
| **ChromaDB** | <100 万条 | 轻量、易用 | 功能有限 |
| **Qdrant** | <1000 万条 | 高性能、功能全 | 需部署服务 |
| **Milvus** | >1000 万条 | 企业级、分布式 | 复杂度高 |

**推荐**: 当前数据规模用 **SQLite + NumPy** 即可

## 五、实现方案

### 5.1 向量化处理脚本

```python
# src/etl/embed_tenders.py
import numpy as np
from FlagEmbedding import BGEM3FlagModel  # 或其他模型

class TenderEmbedder:
    def __init__(self, model_name='BAAI/bge-m3'):
        self.model = BGEM3FlagModel(model_name, use_fp16=True)

    def embed_main(self, tender):
        """生成主向量"""
        text = f"""
项目名称：{tender['project_name']}
产品关键词：{', '.join(tender['product_keywords'])}
应用场景：{tender['application_scenario']}
技术要求：{tender['technical_requirements_summary']}
"""
        return self.model.encode(text)['dense_vecs'][0]

    def embed_batch(self, tenders):
        """批量向量化"""
        texts = []
        for t in tenders:
            text = f"{t['project_name']} {' '.join(t['product_keywords'])} {t['application_scenario']}"
            texts.append(text)

        embeddings = self.model.encode(texts)['dense_vecs']
        return np.array(embeddings)
```

### 5.2 向量检索类

```python
# src/etl/vector_search.py
import numpy as np
import sqlite3
from sklearn.metrics.pairwise import cosine_similarity

class TenderSearch:
    def __init__(self, db_path, embedding_model):
        self.db_path = db_path
        self.model = embedding_model

    def search(self, query, top_k=20, province=None, budget_min=None):
        """
        搜索相似招标公告

        Args:
            query: 查询文本
            top_k: 返回数量
            province: 省份过滤
            budget_min: 最低预算过滤

        Returns:
            list: 匹配的结果
        """
        # 1. 查询向量化
        query_vec = self.model.encode(query)['dense_vecs'][0]

        # 2. 加载向量数据
        conn = sqlite3.connect(self.db_path)

        # 构建 WHERE 条件
        where_clauses = []
        params = []
        if province:
            where_clauses.append("t.province = ?")
            params.append(province)
        if budget_min:
            where_clauses.append("t.budget_amount >= ?")
            params.append(budget_min)

        where_sql = " AND ".join(where_clauses) if where_clauses else "1=1"

        # 查询 tenders 和 embeddings
        sql = f"""
        SELECT t.*, e.embedding_main
        FROM tenders t
        JOIN tender_embeddings e ON t.id = e.tender_id
        WHERE {where_sql}
        """

        cursor = conn.execute(sql, params)
        rows = cursor.fetchall()
        conn.close()

        # 3. 计算相似度
        similarities = []
        for row in rows:
            embedding = np.frombuffer(row[-1], dtype=np.float32)
            sim = cosine_similarity([query_vec], [embedding])[0][0]
            similarities.append(sim)

        # 4. 排序返回 Top-K
        indices = np.argsort(similarities)[::-1][:top_k]

        results = []
        for idx in indices:
            result = dict(zip([d[0] for d in cursor.description[:-1]], rows[idx][:-1]))
            result['similarity'] = float(similarities[idx])
            results.append(result)

        return results
```

### 5.3 使用示例

```python
# 初始化
from FlagEmbedding import BGEM3FlagModel
from tender_search import TenderSearch

model = BGEM3FlagModel('BAAI/bge-m3', use_fp16=True)
searcher = TenderSearch('data/ccgp_data.db', model)

# 简单搜索

for r in results:

# 组合搜索（向量 + 过滤）

    budget_min=1000000
)
```
